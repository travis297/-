# 二手车价格预测项目 - 模型分析总结

## 1. 项目背景与目标
本项目旨在通过对二手车交易数据的分析与建模，预测二手车的交易价格。我们尝试了多种机器学习模型，包括线性回归、树模型以及神经网络，通过对比不同模型的表现，选出最优的预测方案。

## 2. 数据处理与特征工程
在数据预处理阶段，我们进行了以下关键操作：
- **缺失值处理**：针对缺失数据进行了合理的填充。
- **特征编码**：对类别特征进行了 Label Encoding 和 One-Hot Encoding 处理。
- **异常值处理**：识别并处理了部分异常数据点。
- **特征构造**：通过特征交叉和统计特征构造，增强了模型的表达能力。

## 3. 模型性能对比 (MAE)
我们使用 **平均绝对误差 (MAE)** 作为主要评价指标。MAE 越低，模型的预测精度越高。

以下是各模型的最终测试集表现排名：

| 排名 | 模型名称 | 模型类型 | MAE (元) | 评价 |
| :--- | :--- | :--- | :--- | :--- |
| **1** | **Extra Trees (极限随机树)** | **树形模型** | **627.88** | **表现最佳，误差最小。** |
| 2 | LightGBM | 树形模型 | 645.91 | 表现优异，仅次于 Extra Trees，且训练速度较快。 |
| 3 | Random Forest (随机森林) | 树形模型 | 660.52 | 表现稳健，作为强基准模型。 |
| 4 | XGBoost | 树形模型 | 718.74 | 表现良好，但在此数据集上略逊于 LightGBM 和 Extra Trees。 |
| 5 | MLP (神经网络) | 神经网络 | 738.51 | 表现尚可，但在表格数据上未能超越顶级树模型。 |
| 6 | Decision Tree (决策树) | 树形模型 | 1550.20 | 单棵决策树容易过拟合，仅作为基准参考。 |
| 7 | Ridge (岭回归) | 线性模型 | 2599.08 | 线性模型难以捕捉复杂的非线性关系。 |
| 8 | Lasso (套索回归) | 线性模型 | 2659.78 | 同上，表现最差。 |

## 4. 关键发现与分析

### 4.1 树模型的统治地位
在该数据集上，**树形集成模型 (Ensemble Tree Models)** 展现出了压倒性的优势。
- **Extra Trees** 以 627.88 的 MAE 夺得冠军，说明其通过极度随机化的特征选择和阈值划分，有效降低了方差，在该数据集上泛化能力最强。
- **LightGBM** (645.91) 紧随其后，证明了梯度提升决策树 (GBDT) 在处理表格数据时的强大能力。

### 4.2 神经网络的表现
我们尝试了 **MLP (多层感知机)**，使用了 K-Fold Target Encoding 处理类别特征。
- 虽然 MLP (738.51) 的表现优于单棵决策树和线性模型，但未能超越集成树模型。
- 这符合预期，因为在处理中等规模的结构化表格数据时，精心调优的 GBDT 或随机森林通常比全连接神经网络更具优势，且对特征预处理的要求更低。

### 4.3 线性模型的局限性
线性回归模型 (Ridge, Lasso) 的 MAE 均在 2600 左右，远差于非线性模型。这强烈暗示了特征与目标变量之间存在复杂的非线性关系，单纯的线性组合无法有效拟合。

## 5. 结论与建议
1. **推荐模型**：建议使用 **Extra Trees** 作为最终的预测模型，或者尝试将 Extra Trees 与 LightGBM 进行加权融合 (Stacking/Blending)，以进一步提升稳定性。
2. **后续优化方向**：
   - 可以进一步对 Extra Trees 和 LightGBM 进行超参数微调 (Hyperparameter Tuning)。
   - 尝试更多的特征工程，例如业务特征的挖掘。
   - 考虑使用更复杂的模型融合策略。
